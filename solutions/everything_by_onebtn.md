# 버튼 패널 하나로 모든 것을 처리하기
[◀](../README.md)

**이건 사실상 불가능한 방법이고 이론상으로만 적어둔다.**

보통 서버폼에서 버튼을 처리할 때 `메인 패널 -> 팩토리 -> 버튼 패널` 순으로 처리한다.
그리고 어느정도 복잡하게 만들면 
```
메인 패널 
ㄴ 팩토리 -> 버튼 패널1
ㄴ 팩토리 -> 버튼 패널2
ㄴ 팩토리 -> 버튼 패널3
ㄴ 팩토리 -> 버튼 패널4
```
이렇게 된다.

하지만 만약 팩토리를 최대한 줄일 수 있다면?
```
메인 패널 
ㄴ 팩토리
    ㄴ 버튼 패널(1)
    ㄴ 버튼 패널(2)
    ㄴ 버튼 패널(3)
    ㄴ 버튼 패널(4)
```

이렇게 짜는게 가능하다면 확연하게 렉이 줄어들 것이다.
그리고 이론상으로는 가능하다.
**하지만 이걸 시도하면 안 그래도 사람 미치게 하는 json-ui가 더 혐오스럽게 느껴질 수도...**

### 대체 어떻게 가능한거냐?
- 버튼 패널 하나를 기준으로 예외처리로 나누는 것이다.
- 모두 같은 패널이지만 순서, 이름, 텍스쳐에 따라 전부 다르게 요소를 구성하는 것
- 하지만 서버폼의 팩토리는 stack_panel이 기본이기에 이게 전부 한줄로 나열될 것이다.
- 이걸 적절한 위치에 배치하기 위해서 얼마나 많은 고생을 해야할까......
- 나는 못할 듯......
